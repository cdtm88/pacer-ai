<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client -> API (GET export.zwo) | Authenticated request with attacker-controllable session_id path param |
| session row -> ZWO XML text | Session objective/description strings flow into XML element text |
| profile.ftp -> Power fraction | FTP value (or 100W default) is the only authoritative source for power; never LLM output |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Elevation of privilege | GET /sessions/{id}/export.zwo (IDOR) | mitigate | Dual-filter query `.eq("id", session_id).eq("user_id", user_id)`; non-matching session returns 404 (existing PATCH pattern) |
| T-05-02 | Tampering | session_id path param (traversal/injection) | mitigate | `validate_uuid(session_id, "session_id")` before any DB call (V5) |
| T-05-03 | Tampering | session objective/description in XML | mitigate | `xml.etree.ElementTree` auto-escapes element text; no manual string concatenation |
| T-05-04 | Integrity | Power fraction out of range | mitigate | Clamp to 0.0-2.0 in generate_zwo before building any element (ZWO-02); rendered as decimal string |
| T-05-05 | Spoofing | unauthenticated export | mitigate | `Depends(get_current_user)` on the route; user_id sourced only from JWT sub claim |
| T-05-SC | Tampering | npm/pip installs | mitigate | No new package installs in this plan (stdlib ElementTree only); RESEARCH Package Legitimacy Audit lists no backend packages |
</threat_model>
