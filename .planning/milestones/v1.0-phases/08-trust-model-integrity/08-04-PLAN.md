<objective>
Make `generate_plan` actually consume the load/schedule signals it is already handed. Today `current_ctl` and `load_targets` are dead parameters (accepted, never read) and `preferred_days` does not exist — sessions always fall on `_DEFAULT_DAYS` and the 4-week template ignores the user's starting fitness, so the back-protective caps the roadmap promises never constrain anything (CONTEXT.md defect #7 / D-07).

This plan is the pure-computation half of D-07: it adds `preferred_days` scheduling and a CTL-gap-aware progression that flattens volume for genuine low-base beginners and tightens week 1 further when an at-risk beginner (low base + moderate back) is detected. Plan 06 supplies the real server-injected values into these parameters; this plan makes the function honor them, unit-tested via direct calls.

Purpose: PLAN-07 (D-07 computation); satisfies PLAN-06's "back-protective caps actually constrain sessions".
Output: rewired `plan.py` and a new dedicated `tests/sports_science/test_plan.py`.
</objective>
