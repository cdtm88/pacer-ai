<objective>
Close the trust-scanner's substring-attribution bypass (CONTEXT.md defect #3 / D-03). Today `scan_buffer` attributes a bare number if it appears as a raw substring of any tool-result string, so "250" is treated as sourced whenever "2500", "0.250", or a timestamp containing "250" was returned by any tool this turn — a near-total bypass once any tool has run. Replace the substring check with numeric-token extraction (boundary-aware regex) plus a float-tolerance compare, for both Pattern A (number+unit) and Pattern B (unit+number) attribution paths.

Purpose: TRUST-08 — the input-validation control at the heart of this phase; hardens the already-shipped TRUST-03 enforcement under real conversation load.
Output: a rewritten attribution helper in trust.py and extended tests proving the collision class is gone while real attributions still pass.
</objective>
