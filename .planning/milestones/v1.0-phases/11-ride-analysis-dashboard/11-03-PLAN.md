<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → API (`GET /api/rides/{id}/stream`) | Untrusted `ride_id` path param + JWT. Server verifies the JWT `sub`, validates the UUID, and dual-filters ownership. |
| API → Supabase Storage | Route downloads `raw_fit_path` bytes; the object may be missing, stale, or (already size-capped) corrupt. |
| stored bytes → parser | Corrupt bytes must not crash the event loop or the function. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-11-01 | Information Disclosure | get_ride_stream rides SELECT | high | mitigate | Dual `.eq("id", ride_id).eq("user_id", user_id)` filter on the verified JWT `sub`; uniform 404 on any miss (never 403, which would confirm the ride exists for another user). Mirrors sessions.py export_session_zwo exactly. `validate_uuid` rejects malformed IDs (400) before any DB call. |
| T-11-02 | Denial of Service | Storage download | medium | mitigate | 404 when `raw_fit_path` is null (Storage upload is best-effort at upload time); `storage.download()` wrapped in try/except → 404 on any StorageApiError/exception. No new size check needed — only files that passed the 25 MB `MAX_UPLOAD_BYTES` cap can be in Storage (T-11-02 reuses the existing cap). |
| T-11-03 | Denial of Service | parse_fit_stream call | medium | mitigate | Parser uses `fitdecode.ErrorHandling.WARN` and returns `None` on total failure (11-01); the route converts `None` → structured 422; parse runs under `asyncio.to_thread` so a slow parse never blocks the event loop. |
| T-11-SC | Tampering | npm/pip installs | n/a | accept | No new packages this phase (RESEARCH Package Legitimacy Audit: not applicable — every library already installed). No install task, no slopcheck checkpoint required. |
