# Phase 4 — UI Design Contract

> Visual and interaction contract for the React PWA frontend and Google Calendar integration.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn/ui (canary) |
| Preset | CSS-first via `npx shadcn@canary init`; no preset string (Tailwind v4 CSS @theme) |
| Component library | Radix UI (via shadcn/ui) |
| Icon library | lucide-react (bundled with shadcn/ui) |
| Font | Inter (Google Fonts or self-hosted; fallback: ui-sans-serif, system-ui, sans-serif) |

Source: CONTEXT.md D-11, RESEARCH.md Standard Stack, PRD §Design system, CLAUDE.md Technology Stack.

---

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gap, badge padding, inline chip padding |
| sm | 8px | Button inner padding (vertical), list row gap, chat bubble padding |
| md | 16px | Card padding, section heading margin, form field spacing |
| lg | 24px | Card-to-card gap, screen horizontal padding (mobile) |
| xl | 32px | Desktop sidebar item gap, screen section breaks |
| 2xl | 48px | Major screen-level breaks (e.g. Today card to next-few-days list) |
| 3xl | 64px | Desktop content-area top padding |

Exceptions:
- Bottom tab bar height: 56px (touch target compliance; 44px touch target centered in 56px bar)
- iOS safe area: bottom padding on BottomTabBar uses `env(safe-area-inset-bottom)` additional to 56px
- During-Session current-step display: 0 vertical padding (full-bleed font, no card)
- Desktop sidebar width: 240px (fixed; not derived from spacing scale)

---

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Display | 40px | 700 (bold) | 1.1 | During-Session current step target (e.g. "Zone 2 — 20 min") |
| Heading | 20px | 600 (semibold) | 1.2 | Screen titles, Today session card objective, section labels |
| Body | 16px | 400 (regular) | 1.5 | Session descriptions, chat messages, agenda row detail |
| Label | 14px | 500 (medium) | 1.4 | Button text, tab labels, badge text, form chip, nav labels |
| Caption | 12px | 400 (regular) | 1.4 | Timestamps, compliance percentage, muted metadata |

**Hard rules:**
- Inter at all roles. No second typeface.
- Only two weights in the component layer: 400 (regular) and 600 (semibold). 500 and 700 are permitted only in the two roles above (Label and Display) and nowhere else.
- #228BE6 (blue-6) is used for button fills, large interactive elements, and large text only. Never on body (16px) or caption (12px) text directly on white.
- Small blue text (links, active tab label) uses #1B73C0 (blue-7) to meet 4.95:1 contrast on white.
- Body copy color: --ink (#1A2230) primary, --ink-2 (#5F646E) secondary. Never pure black (#000000).

Source: PRD §Design system, CONTEXT.md D-11, REQUIREMENTS.md UI-08.

### 60/30/10 Color Split

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | --color-bg #F9F9FA | App background, screen backgrounds |
| Secondary (30%) | --color-surface #FFFFFF | Cards, sidebar, bottom tab bar, nav, chat bubbles |
| Accent (10%) | --color-blue-6 #228BE6 | Active tab indicator, primary buttons, session card action row, zone-endurance badge, Start Session button fill |
| Destructive | --color-bad #C0341D | "Mark Missed" confirmation text, disconnect calendar confirmation |

**Accent reserved for:**
1. Primary action buttons (Start Session, Connect Google Calendar, Send message in Chat)
2. Active navigation state (active tab indicator, active sidebar item)
3. Zone-endurance intensity badge on Agenda/History rows
4. Onboarding progress indicator
5. Links in chat messages

**Never use accent for:** body text, secondary buttons, card backgrounds, dividers, muted labels.

Source: PRD §Design system, CONTEXT.md D-11/D-12.

## Screen-by-Screen Contract

### Login Screen (`/login`)

**Layout:** Full-screen centered card on --color-bg. Card width: 400px max (desktop), full-width minus 2×lg (mobile).

**Elements:**
- PacerAI logotype (heading weight, --color-ink) centered
- One-line descriptor: "Your adaptive cycling coach." (body, --color-ink-2)
- Email input field (shadcn `<Input>`)
- Primary CTA: "Send magic link" (shadcn `<Button variant="default">`, blue-6 fill)
- Post-submit state: "Check your email" (heading, --color-ink) + "We sent a link to {email}. Click it to sign in." (body, --color-ink-2)
- No password field; no "Sign up" vs "Sign in" distinction

**Empty/error states:**
- Empty email: inline validation "Enter your email address"
- Invalid email format: inline "Enter a valid email address"
- Send failure: sonner toast "Could not send magic link. Try again."

---

### Onboarding Screen (`/onboarding`)

**Layout:** Full-screen, no navigation chrome. White surface. Mobile: full viewport. Desktop: centered column, max-width 640px.

**Elements:**
- Progress bar at top (thin 4px bar, blue-6 fill, width animates as interview advances)
- Chat-style message list: coach messages left-aligned (--color-bg-2 bubble), user responses right-aligned (blue-1 bubble, --color-ink text)
- Input area pinned to bottom: text field + send button (blue-6)
- On profile confirmation: full summary card with section headings (Goals, Schedule, Equipment, Health notes), "This looks right" CTA (blue-6) and "Edit a detail" secondary link

**States:**
- Streaming: coach bubble shows animated ellipsis while tokens arrive
- Profile confirmation: CTA changes to "Start my plan" after user confirms
- Error: sonner toast "Connection lost. Reconnecting..." with auto-retry

---

### Today / Home Screen (`/`)

**Layout (mobile):** Single column. Session card full-width. Next-few-days strip below. Bottom tab bar fixed.

**Layout (desktop):** Two-column. Left: session card + next-few-days. Right: empty in Phase 4 (reserved for Phase 2 fitness panel).

**Session Card:**
- Background: --color-surface, border: 1px --color-line, border-radius: 12px, padding: lg (24px)
- Top line: weekday + date in label weight (--color-ink-2)
- Session objective: heading size (20px, semibold, --color-ink)
- Structure text: body (16px, regular, --color-ink-2) — warm-up / main / cool-down as plain prose
- Targets chip row: RPE chip and/or HR chip and/or power zone chip; zone chip uses zone color (background tint = zone color at 15% opacity, text = zone color darkened for contrast)
- Duration: label (14px, --color-ink-2) with clock icon from lucide-react
- TSB form chip: shown ONLY when `pmc_history.tss_display_ready = true` (28+ days of data). When shown: "fresh", "balanced", or "fatigued" text in a pill — fresh: good green background tint, balanced: blue-0 tint, fatigued: amber background tint. When not shown: no placeholder, no empty space allocated.

**Action row (4 buttons, full-width below session card):**
1. "Start session" — `<Button variant="default">` blue-6 fill, full-width, leading icon: Play from lucide
2. "Export to Zwift" — `<Button variant="outline">` disabled state with tooltip "Coming in the next update", leading icon: Download
3. "Mark done" — `<Button variant="outline">` --color-good text + border, leading icon: CheckCircle
4. "Mark missed" — `<Button variant="ghost">` --color-bad text, leading icon: XCircle

**Next-few-days strip:**
- Horizontal scroll on mobile, vertical list on desktop
- Each row: date label (caption, --color-ink-2) + session type (label, --color-ink) + zone color dot (8px circle) + duration (caption, --color-ink-3)
- No action buttons on future rows (tap only to navigate to Agenda)

**Empty state (no session today):**
- Heading: "No session today"
- Body: "Your next ride is {day}. Rest up."
- No CTA (intentional — rest day is valid state)

**Error state (API failure):**
- Inline error card: "Could not load today's session. Pull down to refresh." with Retry button

### Agenda Screen (`/agenda`)

**Layout:** Single scrollable column. Grouped by week with sticky week header.

**Week header:** "Week of Jun 23" — label weight, --color-ink-2, background: --color-bg (sticky)

**Session row:**
- Left: date column (caption, --color-ink-2), 48px wide
- Center: type label (label, --color-ink) + objective preview (caption, --color-ink-3, 1 line truncated)
- Right: zone color dot (12px circle using zone color) + duration (caption, --color-ink-2)
- Status indicator: checkmark (--color-good) if complete, X (--color-bad) if missed, nothing if upcoming
- Tap to expand: reveals full session plan (objective, structure prose, targets) in an accordion using shadcn `<Accordion>`
- Divider: 1px --color-line between rows

**Zone color dots map exactly to:**
- Recovery: #2B8A5B
- Endurance: #228BE6
- Tempo: #F0A030
- Threshold: #E8590C
- VO2max: #C92A2A

**Empty state:**
- Heading: "No sessions planned yet"
- Body: "Complete the interview in Chat to generate your plan."
- CTA: "Go to Chat" (text button, blue-7)

**Error state:**
- Inline: "Could not load your plan. Tap to retry."

### History Screen (`/history`)

**Layout:** FIT upload zone pinned at top (always visible). Scrollable ride list below.

**FIT Upload Zone:**
- Background: --color-bg-2, border: 2px dashed --color-line, border-radius: 8px, height: 80px
- Default text: "Drop a .FIT file here, or tap to upload" (label, --color-ink-2, centered)
- Icon: Upload from lucide-react, --color-ink-3
- Drag-over state: border color changes to blue-6, background changes to blue-0
- Uploading state: progress spinner (blue-6) + "Uploading ride..."
- Success: sonner toast "Ride uploaded. History updated." (list auto-refetches)
- Error: sonner toast "Upload failed. {error reason}. Try again."

**Ride row:**
- Date (label, --color-ink)
- Compliance chip: "95% on target" (label, --color-good) or "72% on target" (label, --color-warn) or "Unmatched" (label, --color-ink-3) when no planned session
- TSS value: "TSS 78" (caption, --color-ink-2)
- Duration (caption, --color-ink-2)
- Tap to expand: power/HR summary, planned-vs-actual delta table (using shadcn `<Table>`), methodology string from tool result displayed as muted footnote (caption, --color-ink-3)

**CTL sparkline:**
- Shown ONLY when `pmc_history.tss_display_ready = true`
- Renders above the ride list
- Simple line chart (recharts `<LineChart>`), 7-30 day window
- Line color: blue-6, no fill, no axes labels (sparkline format)
- Container height: 48px, full-width
- When not shown: no placeholder

**Empty state (no rides uploaded):**
- Heading: "No rides yet"
- Body: "Upload a .FIT file from Zwift or your head unit to see your history."

**Error state:**
- Inline: "Could not load history. Tap to retry."

### During-Session Screen (`/session`)

**Layout:** Full-screen, no navigation chrome. Dark-ish surface to reduce eye strain: --color-bg-2 background.

**Step hierarchy (Phase 4 — static layout):**
- Current step: Display size (40px bold, --color-ink), "Zone 2 — 20 min" format. Centered. 64px top padding.
- Zone color strip: 4px horizontal bar below current step text, colored with zone color
- Next step: Heading size (20px, semibold, --color-ink-2), "Next: Cool-down 5 min"
- Remaining steps: Body size (16px, regular, --color-ink-3), stacked list, max 3 visible
- Timer display: Display-adjacent — "00:00" in 32px monospace (use `font-variant-numeric: tabular-nums` on Inter). Static/placeholder in Phase 4.
- Timer note: caption "Timer activates in next phase" (--color-ink-3), shown only while static

**Actions:**
- "End session" button: `<Button variant="outline">` positioned bottom-right, --color-bad text

**Phase 4 note:** No ticking timer, no auto-advance. Full layout and typography must be correct so Phase 5 can wire behavior without layout changes.

### Chat Screen (`/chat`)

**Layout:** Full-height flex column. Message list scrollable (fills available height). Input bar pinned at bottom.

**Message bubbles:**
- Coach messages: left-aligned, --color-bg-2 background, --color-ink text, border-radius 12px 12px 12px 4px, padding sm (8px) lg (24px)
- User messages: right-aligned, blue-1 (#CEE5FA) background, --color-ink text, border-radius 12px 12px 4px 12px
- Adaptation reasoning messages: same as coach but with a thin left border in blue-6 (4px) to visually distinguish agent-initiated updates
- Capability gap notes: same as coach but with left border in --color-amber
- Timestamps: caption (12px, --color-ink-3), shown below each bubble

**Streaming state:** Animated ellipsis in an otherwise empty coach bubble while tokens arrive

**Input bar:**
- shadcn `<Textarea>` (single line default, expands to 4 lines max), border: 1px --color-line
- Send button: `<Button size="icon">` blue-6 fill, arrow icon from lucide-react
- Input bar background: --color-surface, top border: 1px --color-line
- Padding: sm (8px) lg (24px)

**Empty state (no conversation yet):**
- Heading: "Ask your coach anything"
- Body: "Start by uploading a ride, or ask about your plan."

**Error state (SSE disconnect):**
- Inline banner above input: "Connection lost. Reconnecting..." (yellow/amber background, --color-warn text, auto-dismisses on reconnect)

### Settings Screen (`/settings`)

**Layout:** Single scrollable column. Grouped by section using visual card separation.

**Sections:**
1. Profile — display name, email (read-only), magic link re-send
2. Google Calendar — status chip + connect/disconnect action
3. Account — "Sign out" destructive button

**Google Calendar section:**
- Connected state: green chip "Connected" (--color-good background tint, --color-good text) + "Disconnect" text button (--color-bad text)
- Disconnected state: "Connect Google Calendar" `<Button variant="outline">` + explanatory caption "Planned sessions will appear in your calendar and update when your plan changes."
- Connecting state: button shows spinner, text "Connecting..."
- Sync failure notice: non-blocking sonner toast (never inline; does not block settings page)

**Destructive confirmation (Mark Missed / Disconnect Calendar):**
- shadcn `<AlertDialog>` (not browser `confirm()`)
- Mark Missed: title "Mark this session as missed?", body "This will trigger a re-plan. Your coach will adjust upcoming sessions.", Confirm: "Yes, mark missed" (--color-bad button), Cancel: "Keep it"
- Disconnect Calendar: title "Disconnect Google Calendar?", body "Future plan changes will no longer sync to your calendar. Existing events are not deleted.", Confirm: "Disconnect" (--color-bad button), Cancel: "Keep connected"

### PWA Install Banner (iOS)

**Trigger:** iOS device (UA check: `/iphone|ipad|ipod/i` + `'ontouchstart' in window`), not already in standalone mode (`!window.matchMedia('(display-mode: standalone)').matches`), not previously dismissed (`localStorage.getItem('ios-banner-dismissed')` is null).

**Position:** Fixed, bottom of screen, above bottom tab bar. `bottom: calc(56px + env(safe-area-inset-bottom))`, `inset-x: lg (16px)`.

**Appearance:** --color-surface background, 1px --color-line border, 12px border-radius, lg (24px) padding, shadow-md.

**Copy:**
- Main text: "Install PacerAI: tap the Share button, then 'Add to Home Screen'." (body, --color-ink)
- Dismiss: "Got it" text button (label, --color-ink-2), right-aligned

**Behavior:** Dismiss writes `localStorage.setItem('ios-banner-dismissed', '1')`. Banner never shown again after dismiss. Also auto-hides if user enters standalone mode on next visit.

## Interaction States

All interactive elements must implement all four states:

| State | Implementation |
|-------|---------------|
| Default | As specified per component above |
| Hover | `bg-bg-2` for ghost/outline buttons; `blue-5` (#3494E8) fill for primary buttons |
| Focus | `ring-2 ring-blue-6 ring-offset-2` (shadcn default focus ring) |
| Disabled | `opacity-50 cursor-not-allowed` (shadcn default); applies to Export to Zwift button |
| Loading | Inline spinner (lucide `<Loader2 className="animate-spin">`) replaces icon; text unchanged |
| Destructive hover | `bg-bad text-white` on confirm buttons |
