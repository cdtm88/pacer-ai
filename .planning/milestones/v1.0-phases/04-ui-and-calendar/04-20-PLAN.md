<objective>
Close UAT GAP 5 (Test 14: "no response" in Chat). Two frontend bugs in api.ts block the chat flow:

1. Shape mismatch: the backend POST /conversations/ returns {conversation_id: "..."} (api/routes/chat.py create_chat_conversation), but createConversation() in api.ts types the response as Conversation and returns res.json() unchanged — so the returned object has no id field. In ChatScreen, the guard that bails when conversation?.id is falsy is therefore always true and handleSend bails before any SSE request is made.

2. Malformed SSE URL: sseUrl() always appends a token query param with a leading ? even when the path already has a query string. ChatScreen passes /chat/stream?conversation_id=...&message=..., so sseUrl produces a URL with two ? characters, which is invalid and breaks the stream. (OnboardingScreen passes a bare /onboarding/start with no query string, so it must keep using ?.)

The 04-13 fix addressed the backend chat_stream message param and turn persistence, but never touched these two api.ts client bugs. This plan fixes both in api.ts only.

Purpose: UI-06 (Chat screen). Sending a message must open the SSE stream and stream back the coach's reply.

Output: createConversation maps conversation_id to id; sseUrl chooses the correct separator based on whether the path already contains a query string. No backend changes.
</objective>
