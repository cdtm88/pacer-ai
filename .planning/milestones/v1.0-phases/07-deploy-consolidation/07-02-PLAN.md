<objective>
Convert the three remaining FastAPI BackgroundTasks call sites (onboarding plan-calendar-sync, and the two adaptation calendar-event updates) to inline-await, applying the exact pattern already proven safe in Phase 6 for backend/routes/rides.py. Vercel freezes/kills the function after the response is sent, so any fire-and-forget BackgroundTask may silently never run in production.

Purpose: Complete the serverless-correctness migration the ROADMAP mandates; after this plan, backend/ has zero BackgroundTasks call sites and all post-response work completes before the HTTP response.
Output: Two converted route modules, two updated test files asserting inline completion, and a bounded timeout confirmed around the Google Calendar calls.
</objective>
