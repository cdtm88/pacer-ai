<objective>
Fix stale-session hijack (item 1, D-06) and live-resume overshoot (item 8, Claude's Discretion), both anchored in the DuringSessionScreen / sessionPersistence pair.

- Item 1 (D-06): PersistedSession has no identity — TodayScreen redirects to /session on ANY persisted entry, so a stale localStorage record from a previous day/session hijacks Today and can mark the wrong session done. Add sessionId + date fields and silently discard on mismatch, rendering Today's real state. No UI (explicitly confirmed no banner/toast/dialog in 09-UI-SPEC.md §3).
- Item 8: the reload path (computeRestoredState) already fast-forwards correctly through multiple elapsed steps, but the live path fires a bare goNext() once per secondsLeft===0, advancing only one step and resetting the new step to full duration — silently absorbing overshoot. Route the live path through the same fast-forward logic.

Purpose: Persisted sessions cannot hijack Today or mark the wrong session done, and backgrounding through several steps resumes at the correct place.
Output: sessionId/date on PersistedSession + mismatch guards; shared fastForwardSteps helper; multi-step-elapsed test coverage.
</objective>
