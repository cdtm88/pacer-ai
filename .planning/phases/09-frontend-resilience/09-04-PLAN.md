<objective>
Harden the auth/router layer (items 10, 11, 12) — the security-relevant cluster of this phase per 09-RESEARCH.md Security Domain (V2 Authentication, V3 Session Management).

- Item 10 (Claude's Discretion / ASVS V3): cross-account cache bleed. router.tsx's onAuthStateChange clears the React Query cache on SIGNED_OUT/USER_UPDATED but NOT on SIGNED_IN, so a new sign-in can render the previous account's cached data. Add exactly 'SIGNED_IN' to the OR-chain (Pitfall 5: do NOT add TOKEN_REFRESHED or INITIAL_SESSION — that would wipe the cache on every silent token refresh).
- Item 11 (Claude's Discretion / ASVS V2): auth callback double-exchange. With detectSessionInUrl:true, the manual exchangeCodeForSession(code) in AuthCallbackScreen races the background exchange; the second attempt fails and bounces a valid session to /login. Delete the manual branch and watch the store instead (useAuth already populates it via onAuthStateChange), with a 5-8s timeout fallback to /login.
- Item 12 (D-09, D-10): no error boundary means a render crash white-screens the whole app. Add React Router's ErrorBoundary route property to each of the 5 AppLayout leaf routes with a shared minimal RouteErrorFallback; the nav shell stays mounted.

Purpose: No cross-account data bleed, no login bounce on valid sessions, no white-screen crashes.
Output: ErrorBoundaryFallback component + per-route wiring; SIGNED_IN cache clear; single-exchange auth callback; auth + router-boundary tests.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| principal transition (sign-in / account switch) | Cached data from a prior authenticated principal must not survive into a new principal's session (ASVS V3) |
| auth callback (untrusted URL params) | The PKCE `code` in the callback URL is single-use; consuming it twice violates the security constraint and can bounce a valid session (ASVS V2) |
| screen render → app shell | A thrown render error must be contained so it cannot expose internals or destroy the whole app shell |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-09-04-01 | Information Disclosure | React Query cache across principals (item 10) | high | mitigate | queryClient.clear() on SIGNED_IN (added to existing SIGNED_OUT/USER_UPDATED) ensures a previous account's cached queries cannot render after a new sign-in (ASVS V3 session-data-non-persistence control) |
| T-09-04-02 | Spoofing / Tampering | PKCE single-use code (item 11) | medium | mitigate | Remove the redundant manual exchangeCodeForSession so the code is consumed exactly once by detectSessionInUrl; eliminates a race that could bounce a valid session and closes a theoretical stale-code replay path against the manual handler (ASVS V2) |
| T-09-04-03 | Information Disclosure | error boundary fallback (item 12) | low | mitigate | RouteErrorFallback renders no error message/stack/ID per D-09; useRouteError() is read but never displayed, so internal error text is not exposed to the client UI |
| T-09-04-SC | Tampering (supply chain) | npm installs | n/a | accept | No new packages; React Router's built-in ErrorBoundary route property used instead of adding react-error-boundary (RESEARCH Don't-Hand-Roll) |
</threat_model>
