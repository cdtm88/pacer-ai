<objective>
Fix upload UX + invalidation (item 14) and the AppLayout height chain (item 9) — both Claude's Discretion, deterministic.

- Item 14: (a) add an indeterminate progress bar while uploading (per 09-UI-SPEC.md §4 — indeterminate, not byte-accurate, because fetch progress is unreliable on iOS Safari and this is bug-fix scope not a transport rewrite); (b) validate the .fit extension on drag-drop (browsers do not enforce the input accept attribute on drops); (c) extend the success invalidation beyond ['rides'] to the full set of affected keys (Pitfall 2: the two PMC keys are named inconsistently — ['pmc','latest'] and ['pmc-history'] — a prefix-match shortcut silently misses the History sparkline).
- Item 9: AppLayout's min-h-screen breaks inner scroll panes, so the chat input does not stay pinned and auto-scroll fails. Swap both wrapping divs to h-dvh (Pitfall 4: use dvh, not h-screen/100vh, matching the codebase's existing DuringSessionScreen 100dvh convention to avoid the iOS 100vh overflow bug).

Purpose: Uploads give clear feedback, reject bad files early, and refresh all dependent views; the app shell scrolls/pins correctly on mobile.
Output: progress bar + drag-drop validation + full invalidation in FitUploadZone; h-dvh AppLayout; new FitUploadZone.test.tsx.
</objective>
