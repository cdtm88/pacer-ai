<objective>
Fix the three api.ts contract/export bugs (items 5, 6, 7 — all Claude's Discretion, deterministic).

- Item 5: the Ride interface uses field names the backend never sends (duration_seconds, avg_power_watts, file_name, distance_m, created_at), so History renders dashes. Align the interface exactly to backend/routes/rides.py list_rides SELECT columns and update RideRow's reads in lockstep. Do NOT add a file_name column/migration — content-addressed storage never captures the original filename (RESEARCH Anti-Pattern); delete the dead "Source: {file_name}" block instead.
- Item 6: exportSessionZwo reads err.error but FastAPI wraps HTTPException(detail={...}) as {detail:{error,detail}}, so the session_not_found branch is unreachable. Parse the real shape, mirroring markSessionMissed/markSessionDone/uploadRide which already do it correctly in the same file.
- Item 7: window.open(blobUrl) fires AFTER awaits, outside the user-gesture window, so iOS Safari popup-blocks it. Restructure so the iOS window handle is acquired synchronously at the top of the handler (before any await), then navigated to the resolved blob URL.

Purpose: History shows real numbers; ZWO export errors are actionable; iOS export is not popup-blocked.
Output: aligned Ride interface + RideRow; correct ZWO error parsing; gesture-safe export ordering; extended history/zwo tests.
</objective>
