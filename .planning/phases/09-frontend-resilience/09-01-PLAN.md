<objective>
Fix the two chat-stream resilience bugs and build the shared error banner both chat and onboarding will use.

- Item 2 (D-02): a chat SSE stream error currently sets a permanent "Reconnecting..." banner, never retries, and never clears activeStreamUrl, so the input stays disabled forever. Add a silent 1-2x retry-with-backoff inside useSSEStream; only surface a terminal error (via StreamErrorBanner + manual Retry) after retries are exhausted.
- Item 3 (D-03): a tool-only turn produces isDone=true with empty content, so ChatScreen's commit effect never runs and activeStreamUrl never clears, silently bricking every later send. Split the effect so activeStreamUrl always clears on isDone regardless of content, and only push a coach message when content is non-empty.

This plan also creates the shared StreamErrorBanner component (per 09-UI-SPEC.md §1) that plan 09-06 (onboarding) reuses.

Purpose: Chat input never gets permanently bricked by a transient stream fault or a tool-only turn.
Output: StreamErrorBanner component; retry-with-backoff useSSEStream; corrected ChatScreen done-handling; updated tests replacing the stale "Reconnecting..." assertion (Pitfall 3).
</objective>
