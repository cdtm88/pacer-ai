<objective>
Implement the two highest-risk numerical functions: `sports_science/metrics.py` (`compute_tss`, TOOL-04, with the private `_compute_np` helper) and `sports_science/pmc.py` (`update_pmc`, TOOL-05). These have subtle, well-documented failure modes — NP zeros exclusion, spike-filter ordering, and PMC cold-start noise — so they are written test-first with the edge cases (TOOL-10) encoded as failing tests before any implementation.

Purpose: TSS and PMC are the load and form metrics every downstream coaching decision depends on. If NP is computed as average power, or TSB leaks before 28 days, the whole adaptive loop is built on wrong numbers. The tests here are the trust anchor's correctness proof.
Output: Tested `compute_tss` and `update_pmc`, both returning ToolResult, both sourcing constants only.
</objective>

    `sports_science/metrics.py`:
    - `_compute_np(power_array, ftp) -> float | None`: convert to numpy float array; clip to [0, ftp*NP_SPIKE_MULTIPLIER] (or NP_SPIKE_FALLBACK_WATTS if ftp is falsy) BEFORE any averaging; if len < 30-sample window return None; compute 30-sample rolling mean via numpy convolution (mode='valid'); return float(mean(rolling_mean**4)**0.25). Zeros stay in the array (Pitfall 1).
    - `compute_tss(power_array, duration_secs, ftp) -> ToolResult`: if duration_secs < NP_MIN_DURATION_SECS return ToolResult(value=None, unit="TSS", methodology naming the short-ride reason, inputs=...); compute NP; if NP is None return value=None; IF = NP/ftp; TSS = (duration_secs * NP * IF) / (ftp * 3600) * 100; if duration_secs > 3600 and IF > 1.05 append a data-quality warning string (Claude's Discretion in CONTEXT.md); return ToolResult(value={tss, np_watts, if, warnings}, unit="TSS", methodology="TrainingPeaks TSS = (duration * NP * IF) / (FTP * 3600) * 100", inputs=...).

    `sports_science/pmc.py`:
    - module-level CTL_ALPHA = 1 - np.exp(-1/CTL_TC), ATL_ALPHA = 1 - np.exp(-1/ATL_TC)

Round CTL/ATL/TSB to 2 decimals.
