<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → POST /rides/upload | Untrusted file bytes + user_id + filename cross into the parser and Storage |
| file bytes → fitdecode parser | Malformed/malicious .FIT could exhaust memory or crash the parser |
| backend → Supabase Storage + rides/pmc_history | Writes use SERVICE_ROLE_KEY |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-11 | Denial of Service | file upload size | mitigate | Reject uploads over 25 MB with 422 before parsing; parse runs in a bounded thread; rides < ~2h have ~7200 records (RESEARCH Open Question 1) |
| T-03-12 | Tampering | malformed .FIT input | mitigate | fitdecode ErrorHandling.WARN + duration_secs < 600 guard returns structured 422; never silently succeeds with zeros (D-14) |
| T-03-13 | Information Disclosure | filename path traversal | mitigate | Sanitize filename before building the Storage path fits/{user_id}/{name}; never use raw client filename in a filesystem path |
| T-03-14 | Information Disclosure | cross-user ride data | mitigate | rides and pmc_history have RLS user-owns-row (from 0001); backend inserts scoped to the supplied user_id; reads go through RLS |
| T-03-15 | Tampering | ftp_used audit trail | mitigate | Cold-start TSS records ftp_used=150.0 so estimated-FTP TSS is auditable and backfillable (Risk 1) |
| T-03-SC | Tampering | fitdecode usage | accept | fitdecode==0.11.0 already vetted and installed in 03-02 (CLAUDE.md-mandated); no new install in this plan |
</threat_model>
