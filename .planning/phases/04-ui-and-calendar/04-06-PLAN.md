## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → SSE endpoints | EventSource cannot send headers; JWT travels in ?token= |
| agent text → DOM | Streamed chat content is rendered into the DOM |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-17 | Spoofing | SSE ?token= auth | accept | Documented tradeoff (Open Question 1): short-lived JWT in query string for EventSource; refreshed by supabase-js; token suppressed from logs |
| T-04-18 | Tampering | FIT upload | mitigate | Server-side path-traversal sanitization already in rides.py; frontend sends only the file field; size/duration validated server-side |
| T-04-19 | Tampering (XSS) | ChatBubble rendering streamed content | mitigate | Never use dangerouslySetInnerHTML; render via React safe text rendering (UI-SPEC + Security Domain known pattern) |
| T-04-20 | Information Disclosure | conversation history | mitigate | Conversations are JWT-scoped server-side; ChatScreen reads only the authenticated user's conversation |
</threat_model>
