## Two Bugs Fixed (not yet re-run)

**Bug 1 — PATCH route bypassed mocks:** The `/sessions/[^/]+$/` handler called `route.continue()` for non-PATCH requests. In Playwright, `route.continue()` sends to the real network. Fixed to `route.fallback()` which falls through to the next matching mock handler.

**Bug 2 — Settings gear selector:** AppLayout renders `<button aria-label="Settings">` not a link. Fixed selector to `getByRole('button', { name: 'Settings' })`.

## Architecture Notes

Auth mock strategy in `frontend/tests/e2e/phase4.spec.ts`:
1. `injectAuth(page)` sets `sb-test-pacer-auth-token` in localStorage via `addInitScript()`
2. `interceptSupabaseAuth(page)` intercepts `*.supabase.co/auth/v1/*` calls
3. `mockBackendApis(page, overrides)` intercepts all API paths with fixtures

Critical: Routes are LIFO in Playwright. The PATCH-only handler uses `route.fallback()` so non-PATCH requests fall through to specific `/sessions/today` and `/sessions/upcoming` handlers.
