<objective>
Create the `api/auth.py` JWT dependency and replace every `# SECURITY TODO (Phase 4)` insecure user_id pattern across the four existing route modules with `Depends(get_current_user)`. This is the security gate for the entire API and the auth foundation for the whole phase: until this lands, any caller can supply an arbitrary user_id to read or mutate another user's data, and the new endpoints in 04-01 have no auth module to import.

Purpose: D-03 mandates Supabase JWT verification on every API request, with the Supabase user_id flowing through as the authenticated identity. This is the highest-severity security work in the phase. This plan runs in wave 1 (no dependencies) so that `api/auth.py` exists before plan 04-01's new read endpoints import it.
Output: `api/auth.py` exporting `get_current_user` (header or ?token=); chat.py, onboarding.py, rides.py, adaptations.py all using `Depends(get_current_user)`; SSE endpoints accepting `?token=`; `tests/api/test_auth.py` proving 401 on unauthenticated access and cross-user isolation.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → FastAPI | All four existing route modules cross here; identity must derive from a verified JWT |
| EventSource → SSE endpoints | EventSource cannot send Authorization headers; JWT travels in ?token= query string |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-02 | Spoofing | api/auth.py JWT decode | mitigate | jwt.decode with HS256 + audience='authenticated'; SUPABASE_JWT_SECRET from env, never the anon key (Pitfall 6) |
| T-04-04 | Spoofing | all existing routes (user_id from request) | mitigate | Replace every user_id query/body/form param with Depends(get_current_user); identity comes only from the JWT sub claim |
| T-04-SC | Tampering | PyJWT install | mitigate | slopcheck + legitimacy: PyJWT is RESEARCH.md-approved (github.com/jpadilla/pyjwt, 10+ yrs); no [ASSUMED]/[SUS] blocker |
| T-04-05 | Information Disclosure | EventSource JWT in ?token= | accept | Documented tradeoff (Open Question 1): query-param token is simplest for EventSource; access tokens are short-lived (Supabase ~1hr) and refreshed by @supabase/supabase-js; suppress `token` from server logs in deployment config |
| T-04-06 | Elevation of Privilege | rides upload, adaptations mutate | mitigate | user_id sourced from JWT so a caller cannot upload a ride or trigger a replan as another user |
| T-04-07 | Repudiation | mark_session_missed | mitigate | Authenticated identity is logged in adaptations audit trail; no anonymous mutations |
</threat_model>
