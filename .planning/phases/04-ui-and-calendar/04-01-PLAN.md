<objective>
Add the Phase 4 DB migration and the six backend read/create endpoints the frontend cannot function without. This is the data-layer foundation: the migration reconciles column-name drift from Phase 3 and adds calendar tracking columns; the new endpoints expose session, ride, PMC, profile, and conversation data that no existing route provides.

Purpose: Every frontend screen (Today, Agenda, History, Chat, Settings, first-run gate) reads from endpoints that do not yet exist. Without this plan, the frontend has nothing to render.
Output: `supabase/migrations/0003_phase4_schema.sql` applied to the live DB; `api/routes/sessions.py` with 4 GET endpoints; `GET /rides/` added to rides.py; `POST /conversations/` added to chat.py; tests in `tests/api/test_sessions.py`.

This plan does NOT add JWT auth (plan 04-02 does that). These new endpoints are written from the start using `Depends(get_current_user)` as a forward reference — see note in Task 2. To avoid a circular dependency with 04-02, this plan creates `api/auth.py` with the `get_current_user` dependency as part of Task 2's prerequisite, and 04-02 then migrates the EXISTING routes to use it.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → FastAPI | Untrusted HTTP requests cross here; user_id must come from a verified JWT, never request input |
| FastAPI → Supabase | Service-role key bypasses RLS; application-layer user_id filter is the access-control gate |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Elevation of Privilege | new read endpoints in sessions.py/rides.py | mitigate | All handlers use Depends(get_current_user); user_id is taken from the JWT sub claim, never from query/body |
| T-04-02 | Spoofing | api/auth.py JWT decode | mitigate | jwt.decode with HS256 + audience='authenticated'; SUPABASE_JWT_SECRET from env, never the anon key (Pitfall 6) |
| T-04-03 | Information Disclosure | GET /profiles/me, /sessions/* | mitigate | Supabase queries filter `.eq("user_id", user_id)` from the JWT so cross-user reads return no rows |
| T-04-SC | Tampering | PyJWT install | mitigate | slopcheck + legitimacy: PyJWT is RESEARCH.md-approved (github.com/jpadilla/pyjwt, 10+ yrs); no [ASSUMED]/[SUS] blocker |
</threat_model>
