<objective>
Build the trust-model response scanner (`agent/trust.py`, TRUST-03/04/05) and the FastAPI SSE transport (`api/main.py`, `api/routes/chat.py`, AGENT-05). The trust scanner uses a heuristic regex (D-09) to catch unsourced physiological numbers, treats numbers that appear in this turn's tool_result values as attributed, and on a genuine violation logs a capability gap and signals the loop to retry. The SSE route drives `run_turn` from Plan 02 and emits typed `text/event-stream` frames (D-07, D-08) that an EventSource client can consume.

Purpose: This closes the trust loop end-to-end — the scanner is the enforcement mechanism that makes "the LLM never emits an unsourced physiological number" provable, and the SSE endpoint is the only HTTP surface for Phase 2. The loop's `trust_scanner` hook (Plan 02) is satisfied by `scan_buffer` here.
Output: `agent/trust.py` (scan_buffer + on-violation capability-gap hook), `api/main.py` (FastAPI app), `api/routes/chat.py` (GET /chat/stream SSE).
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| assistant text → SSE stream | Buffered text is scanned before any frame leaves the server; unsourced numbers must be blocked |
| client HTTP → /chat/stream | Untrusted conversation_id query param crosses here; user-supplied prompt drives the loop |
| backend → Anthropic API | ANTHROPIC_API_KEY is read from env and must never appear in an SSE frame |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-07 | Tampering | agent/trust.py scan_buffer | mitigate | Regex detects numeric+physiological-unit text; attribution check requires the value to appear verbatim in a tool_result this turn, else it is a violation that triggers retry + capability-gap log (TRUST-03) |
| T-02-08 | Information disclosure | api/routes/chat.py | mitigate | ANTHROPIC_API_KEY read from env only, never logged, never placed in an SSE data frame; error frames carry a generic code, not raw secrets |
| T-02-09 | Input validation | api/routes/chat.py conversation_id | mitigate | conversation_id typed via FastAPI Query(...) (V5 Input Validation); Phase 2 does not use it for DB lookup so no injection surface; tool inputs validated by the Anthropic SDK schema |
| T-02-10 | Denial of Service | api/routes/chat.py SSE | accept | MAX_RETRIES cap in run_turn bounds the turn; proxy idle-timeout mitigation (X-Accel-Buffering: no) documented; keepalive tuning deferred — low risk for local dev (auth + deploy hardening is Phase 4) |
</threat_model>
