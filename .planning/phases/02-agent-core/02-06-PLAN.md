<objective>
Close the open verification gap on the trust scanner's accuracy: RESEARCH.md Open Question 2 ("trust scanner false positive rate — unknown until tested"), Assumption A6 ("attribution via string containment is sufficient"), and Pitfall 1 ("false positives on tool-result echo") are all flagged as unresolved, and Phase 2 Success Criterion 5 explicitly requires proof that the agent does not emit unsourced physiological numbers "across a representative set of scenarios." The existing tests (02-03 Task 1, 02-04 Task 2) assert only a few hand-picked strings, which does not characterize the scanner. This plan adds a labelled representative corpus (`tests/agent/fixtures/trust_corpus.py`) and a parametrized characterization test (`tests/agent/test_trust_corpus.py`) that drives the REAL `scan_buffer` (agent/trust.py, Plan 03) over the corpus and asserts: zero false negatives on the unsourced-violation set (TRUST-03 catches every planted number), zero false positives on the qualitative-coaching set, and correct attribution on the tool-echo set (Pitfall 1).

Purpose: This is the durable proof of Phase 2 Success Criterion 5 and the AGENT-06 representativeness requirement. The scanner regex is Claude's-discretion (D-09); this corpus is the regression harness that lets it be tuned safely — any future loosening that introduces a false negative, or tightening that introduces a false positive, fails here.
Output: `tests/agent/fixtures/trust_corpus.py` (labelled VIOLATIONS / ATTRIBUTED / QUALITATIVE sets), `tests/agent/test_trust_corpus.py` (parametrized scan_buffer characterization).
</objective>
