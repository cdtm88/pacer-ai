## Interaction Contracts

### Visual Hierarchy — DuringSessionScreen

Primary focal point: the current step label (40px bold, `--color-ink`) is the dominant visual anchor. The timer (40px bold, tabular-nums) is the secondary focal point, positioned below the step label. "Next:" (20px bold, `--color-ink-2`) is tertiary. "Skip step" button and remaining steps are subordinate chrome. This stacking order governs layout priority on all viewport sizes.

### Timer Auto-Advance

1. Timer counts down from `step.duration * 60` seconds using `Date.now()` deltas stored in a `startTimestamp` ref. Not `setInterval` counts (IOS-02).
2. On `visibilitychange` (tab returns to foreground): resync elapsed from `Date.now() - startTimestamp`. Do not reset timer.
3. At `elapsed >= stepDuration - 3`: begin countdown warning. Show "Starting [next step] in 3..." replacing sublabel. Count to 1 at 1-second intervals.
4. At `elapsed >= stepDuration`: auto-advance `currentIndex`. If `currentIndex + 1 >= steps.length`, show "Session complete" overlay instead.
5. Skip button taps trigger immediate advance (same logic as timer hitting 0, without countdown).

### Wake Lock

1. On component mount: `await navigator.wakeLock.request('screen')`. If `navigator.wakeLock` is undefined or throws, activate `NoSleep.js` fallback (`new NoSleep(); noSleep.enable()`).
2. On `visibilitychange` (hidden): wake lock releases automatically per spec. Re-request on return to visible.
3. On component unmount or "End session": release wake lock and disable NoSleep.
4. Wake lock failure is silent — no toast, no error state shown to user. The timer continues to function; only the wake lock is skipped.

### ZWO Export Modal

1. "Export to Zwift" button (Today screen) opens a shadcn Dialog modal.
2. Modal fetches session metadata from already-cached TanStack Query data — no extra request to open modal.
3. "Download .zwo" button triggers `GET /sessions/{id}/export.zwo` via `apiFetch` with `Accept: application/octet-stream`. On success, creates a blob URL and triggers browser download. On failure, shows sonner error toast; modal stays open.
4. Modal closes on "Close" button or after successful download.

### Duration Picker Modal

1. "Ride anyway" button (Today rest-day state) opens a shadcn Dialog modal.
2. Preset buttons (30/45/60 min) are mutually exclusive — selecting one highlights it and pre-fills the value.
3. Custom input accepts integers 10–180 (validation: must be >= 10 and <= 180). Show inline error "Enter a time between 10 and 180 minutes" if out of range.
4. "Start session" button is disabled if no duration is selected or custom input is invalid.
5. On "Start session": write duration to `freeRideDurationMins` Zustand slice, navigate to `/session`.
