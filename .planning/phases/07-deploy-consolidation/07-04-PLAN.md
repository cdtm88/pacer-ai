<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| public internet -> vercel.json routing table | Untrusted requests are dispatched to either the static service or the Python function based on this config; a misroute can expose backend source or route static paths through Python |
| Vercel env store -> Python function | Production secrets (SUPABASE_SERVICE_ROLE_KEY, CALENDAR_FERNET_KEY) must exist in Vercel before Railway is decommissioned |
| abandoned Railway deployment -> internet | A still-running Railway service remains publicly reachable and holds a copy of the service-role key |
| browser EventSource -> SSE endpoint | JWT is passed via ?token= query param (existing, out of scope for this phase) |
