<objective>
Fix the production save_profile column mismatch. `backend/sports_science/profile.py`'s
`save_profile()` builds its Supabase upsert dict with the key `"fitness_goals"` (line 92),
but the `public.profiles` table has no such column, the real column is named `goals` (jsonb).
Every other key in the dict matches a real column, so PostgREST rejects every save_profile
call with `PGRST204: Could not find the 'fitness_goals' column of 'profiles' in the schema cache`
(confirmed live in the SSE tool_result stream during a real onboarding session).

The fix is exactly one dict key rename: `"fitness_goals": fitness_goals,` -> `"goals": fitness_goals,`.
Do NOT rename the function parameter `fitness_goals` (the Python API), and do NOT touch the
`fitness_goals` LLM tool-argument name in `backend/agent/tools.py` (correct and unrelated).
`goals` is a jsonb column and the Supabase Python client auto-serializes the existing
`fitness_goals: str` value into valid JSON, so no other change is needed.

Purpose: Unblock onboarding profile persistence in production, every save_profile call
currently fails.
Output: Corrected profile.py, affected pytest suites green, fix pushed to main.
</objective>
